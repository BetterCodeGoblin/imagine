# IMAGINE Gameplay Pillars

## Pillar 1: Effort Must Matter
Player progress should be earned through sustained action, not passive timers or shallow button spam.

## Pillar 2: Motion Should Read Clearly
The player's movement, exertion, and state should be legible moment to moment.

## Pillar 3: Systems Should Reinforce the Theme
Burden, ascent, fatigue, momentum, and recovery should show up as mechanics, not just flavor text.

## Pillar 4: One Vertical Slice First
The project should prove one compelling playable loop before expanding into multiple exercise modes or hardware paths.

## Pillar 5: Hardware Is a Force Multiplier, Not the Foundation
Real-world exercise integration is valuable, but the game still needs to work as a game before specialized hardware support expands.
